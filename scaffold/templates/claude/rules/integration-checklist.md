# 跨端联调检查模板

{{#has_hardware}}
## 数据流地图（部署前必读）

梳理系统中所有数据流向，每条标注协议和关键接口：

| 方向 | 协议 | 关键接口 | 说明 |
|------|------|---------|------|
| 平台 → 设备 | WebSocket | model_update | 模型推送 |
| 设备 → 平台 | HTTP POST | /devices/heartbeat | 心跳上报 |
| 设备 → 平台 | HTTP POST | /devices/stats | 统计上报 |
| ... | ... | ... | ... |

联调时按此图逐条验证。

## 联调必验清单

- [ ] 平台 → 设备：下发指令，设备收到并执行
- [ ] 设备 → 平台：上报数据，平台正确显示
- [ ] 断线恢复：停设备 → 平台标记离线 → 重启 → 自动恢复
- [ ] 数据一致：平台显示的和设备上报的一致（字段名、数值、时间）
{{/has_hardware}}

## 前后端联调必验

- [ ] 前端每个页面加载时调的 API 都存在且返回正确格式
- [ ] 前端每个按钮调的 API 都存在且功能正确
- [ ] 前端显示的字段名和后端返回的字段名完全一致
- [ ] 写操作后前端重新 fetch 了关联数据
- [ ] 刷新页面后状态保持（数据从 API 恢复，不是内存残留）
